================================================================================
 TOOL — check_red_flags   (webhook, READ)
 Tests a quote against the vertical's red-flag rules so the agent can push back.
================================================================================
Mirrors backend/app/engines/red_flag.py and the per-category red_flags config
(e.g. below_market_30 @ 0.70, service_charge_undisclosed, deposit_over_50pct,
incomplete_itemization).

WHY IT EXISTS
-------------
The brief is explicit: a quote 30%+ below market is a WARNING, not a win. This tool
lets the agent sanity-check a number mid-call and, if it's suspiciously low or
under-itemized, probe instead of celebrating.

ELEVENLABS WEBHOOK TOOL CONFIG
------------------------------
name:        check_red_flags
description: Check the current quote against market red-flag rules (e.g. 30%+ below
             market, undisclosed service charge, oversized deposit, incomplete
             itemization). Call once you have a total. If it flags below-market, do
             NOT celebrate — probe what's missing.
method:      POST
url:         {{server_base_url}}/api/agent-tools/red-flags
headers:     Authorization: Bearer <AGENT_TOOLS_SECRET>
             Content-Type: application/json
body params:
  campaign_id   string  required = {{campaign_id}}
  call_id       string  required = {{call_id}}
  category      string  required
  segment_key   string  required
  quote         object  required  { line_items[], terms{}, normalized_per_unit }
response_timeout: 8s

SERVER RESPONSE
---------------
{
  "flags": [
    { "rule_key": "below_market_30", "severity": "high",
      "detail": "Quote is 34% below the segment median — likely missing fees." },
    { "rule_key": "incomplete_itemization", "severity": "medium",
      "detail": "No delivery or setup line item present." }
  ],
  "market_median_per_unit": 95.0    // for the agent's framing, e.g. "I'm hearing ~$95/guest"
}
Empty flags array = quote looks clean.

SERVER CONTRACT
---------------
- Look up the segment benchmark median, run red_flag.evaluate, persist RedFlag rows
  for the quote, and optionally bus.publish(campaign_id, "redflag", {...}) so the War
  Room can badge the vendor live.
- Read-mostly: it evaluates and records; it does not change the price.

HOW THE AGENT USES THE RESULT
-----------------------------
- below_market_30 -> "That's well under what I'm hearing elsewhere. What's usually
  included that isn't here?" Then log the fees that surface.
- incomplete_itemization -> ask directly for the missing line items.
- deposit_over_50pct -> "A 60% deposit is steep — is that refundable, and until when?"
