================================================================================
 TOOL — check_red_flags   (webhook, POST, READ-mostly)
 Tests a quote against the vertical's red-flag rules so the agent can push back.
================================================================================
Mirrors backend/app/engines/red_flag.py and the per-category red_flags config
(below_market_30 @ 0.70, service_charge_undisclosed, deposit_over_50pct,
incomplete_itemization). Read tools/00_tool_config_reference.txt first.

WHY IT EXISTS
-------------
The brief is explicit: a quote 30%+ below market is a WARNING, not a win. This lets
the agent sanity-check a number mid-call and, if it's suspiciously low or
under-itemized, PROBE instead of celebrating.

WHEN TO CALL
------------
Once it has a total (after the opening price, and again after fees are revealed).

-----------------------

SERVER RESPONSE
---------------
{
  "flags": [
    { "rule_key": "below_market_30", "severity": "high",
      "detail": "Quote is 34% below the segment median — likely missing fees." },
    { "rule_key": "incomplete_itemization", "severity": "medium",
      "detail": "No delivery or setup line item present." }
  ],
  "market_median_per_unit": 95.0    // for framing, e.g. "I'm hearing ~$95/guest elsewhere"
}
Empty flags array = quote looks clean.

SERVER CONTRACT
---------------
- Look up the segment benchmark median, run red_flag.evaluate, persist RedFlag rows.
- Optionally bus.publish(campaign_id, "redflag", {...}) so the War Room badges the
  vendor live.
- Read-mostly: it evaluates and records; it does not change the price.

HOW THE AGENT USES THE RESULT
-----------------------------
- below_market_30 -> "That's well under what I'm hearing elsewhere. What's usually
  included that isn't here?" Then log the fees that surface.
- incomplete_itemization -> ask directly for the missing line items.
- deposit_over_50pct -> "A 60% deposit is steep — is that refundable, and until when?"
