================================================================================
 TOOL — log_quote   (webhook, POST, WRITE)
 Persists the current structured, itemized quote and pushes it to the War Room.
================================================================================
Publishes bus events  quote.new  (first time) and  quote.update  (every change),
exactly the events the live dashboard already consumes (see caller.py _quote_event).
Read tools/00_tool_config_reference.txt first.

WHEN TO CALL
------------
Any time a price, line item, fee, or term is learned or changes: the opening number,
each itemized fee, after a price move, after fees are revealed. Always send the FULL
current quote (not a diff) so the server holds the complete latest picture.

SERVER CONTRACT
---------------
- Validate campaign_id exists and the call belongs to it; reject otherwise.
- Upsert the Quote row (models.Quote) for (campaign_id, call_id, vendor).
- Recompute normalized_per_unit (per guest / per hour / per event) for comparability.
- bus.publish(campaign_id, "quote.new" | "quote.update", { quote_id, call_id,
    vendor_id, vendor_name, category, segment_key, opening_total, total, line_items }).
Return: { "ok": true, "quote_id": "...", "total": 8700, "normalized_per_unit": 72.5 }

NOTES
-----
- Keep fees as SEPARATE line items with mandatory/included flags — comparability and
  red-flag detection both depend on itemization, not a lump sum.
- disclosed_voluntarily=false marks a fee the vendor admitted only when asked — the
  Receipt flags those.
- First log_quote of a call = the OPENING quote; the server records opening_total so
  the final report can show the negotiated delta.
