================================================================================
 TOOL — record_price_move   (webhook, POST, WRITE)   ★ the realtime "deal changed" signal
 The single most important live event: a price/term moved because of a lever.
================================================================================
Publishes bus event  price.move  — makes the War Room ticker jump and proves, on the
record, that a number changed DURING the call because of leverage the agent gathered
(a hard success criterion of the challenge).
Mirrors the PriceEvent write + "price.move" publish in caller.py :: _negotiate.
Read tools/00_tool_config_reference.txt first.

WHEN TO CALL
------------
The instant the vendor lowers a total, drops a fee, or changes a term in response to
a lever the agent just used. Call this FIRST (for the live ticker), then log_quote
with the new itemized numbers.

PASTE-READY tool_config
-----------------------
{
  "tool_config": {

    "description": "Record that the price or a term just changed because of a specific lever I used. Call this the instant the vendor moves, before updating the full quote. Attribute the move to exactly one lever_key that I actually just used and that get_verified_leverage returned. Do not call if nothing changed.",
    "response_timeout_secs": 8,
    "api_schema": {

}

SERVER CONTRACT
---------------
- Validate campaign_id/call_id; reject if leverage_key isn't a real lever previously
  offered by get_verified_leverage for this call (guards fabricated drama).
- Reject if to_total == from_total (a "move" of zero is not a move).
- Insert a PriceEvent (from_total, to_total, leverage_type, attributed=true).
- Update SegmentObservation (applied_count / moved_count / sum_delta_pct) so the
  system learns which levers actually work per segment — same as the simulation.
- bus.publish(campaign_id, "price.move", { call_id, vendor_name, category,
    from_total, to_total, leverage, delta_pct }).
Return: { "ok": true, "delta_pct": -5.4 }

HONESTY GUARD
-------------
- Attribute to exactly ONE lever — the one just used. No lever, no move: if the
  vendor drops price for no reason the agent gave, that's suspicious; don't
  manufacture a cause. The prompt tells the agent this too.
